{# Related-content resources section. resources_section(articles, heading='Related content', view_all_url='/blog/', view_all_label='View all posts →') Renders a left-justified H2 heading with a "View all posts →" link on the right, then a 3-column grid of blog-post cards. Each card has a rounded featured image and a title below; the whole card is a single link to the post. Importance flows left to right — the macro renders `articles` in the order it receives them. Parameters ---------- articles : List of dicts, each with: url : Blog post URL (e.g. '/blog/post-slug/'). title : Post title. image_src : Path to the featured image. image_alt : Alt text for the image. heading : Section heading. Default 'Related content'. view_all_url : URL for the "View all posts" link. Default '/blog/'. view_all_label : Text for the view-all link. Default 'View all posts →'. Filtering logic is intentionally OUT of scope for this macro — the caller passes whatever articles should appear. Dynamic filtering by industry tag (Insurance, Real Estate, etc.) with a most-recent fallback is a planned follow-up that requires changes to pelicanconf.py. See COMPONENTS.md "Open questions". Example ------- {% import 'macros/resources.html' as ui_res %} {{ ui_res.resources_section( articles=[ { 'url': '/blog/roofing-permit-data-insurance-roof-age/', 'title': 'The Roof Age Problem: What Permit Data Reveals', 'image_src': '/images/blog_images/roofing-permit-data-insurance-roof-age-hero.png', 'image_alt': 'Aerial view of a neighborhood', }, ]) }} #} {% macro resources_section(articles, heading='Related content', view_all_url='/blog/', view_all_label='View all posts →', wrapper_class='') %}
{# Section header — H2 on the left, "View all posts" link on the right. #}

{{ heading }}

{{ view_all_label }}
{# 3-up card grid — importance left to right. #}
{% for article in articles %}

{{ article.title }}

 {% endfor %}
{% endmacro %}
