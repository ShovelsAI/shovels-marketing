{# Use Cases section — eyebrow + heading + alternating two-column rows. use_case_section(eyebrow, heading, cases, intro=None) Renders a full "Use Cases" section for an Industry page. Each item in `cases` is a dict with the keys: number : '01', '02', etc. Displayed as a small label above the title. title : The use case heading. description : One- or two-sentence body paragraph. bullets : List of bullet strings (optional). Each renders with a green checkmark. image_src : Path to the screenshot or illustration. Pass an empty string (or omit) when the image has not been delivered yet — the macro renders a red "TBD" placeholder in its place and shows image_alt as the intended-image caption. media : Optional pre-rendered HTML for the visual column (e.g. a coded `code_window` captured with {% set %}…{% endset %}). When present it replaces image_src/TBD entirely. Used by the Solutions pages to show code/JSON instead of images. image_alt : Alt text for the image. Doubles as the description shown inside the TBD placeholder when image_src is empty. framed : Optional boolean. Default True (image rendered inside the browser_frame chrome). Set False for non-screenshot imagery like illustrations — renders the image directly with no chrome. caption : Optional small italic caption rendered below the image (e.g., "names, and dates"). image_max_width : Optional CSS max-width applied to unframed images (when framed=False). Defaults to '420px' — the canonical illustration width, applied automatically so illustrations read consistently across pages. Accepts any CSS length, e.g. '612px', to override for genuine exceptions. Ignored when framed=True (browser_frame has its own max_width). The first case displays image-right; the second image-left; alternating thereafter. Alternation is handled internally via loop.index.
