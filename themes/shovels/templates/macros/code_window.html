{# window_header(title='', copyable=False, tone='dark') code_window(title='', copyable=False) Shared "window" chrome for the Solutions pages — a header bar with traffic-light dots, an optional label, and an optional copy button. `window_header` renders just the bar so bespoke cards (geo-resolution card, endpoint list, permit record) can share the same framing as the dark code windows. `code_window` is the dark terminal specialization: header + a monospace 
 body authored via a {% call %} block.

  window_header parameters
  ------------------------
  title    : Optional label shown next to the dots (e.g. 'permits/search').
  copyable : When True, render a copy button (Alpine-wired). Requires the
             body to expose `x-ref="code"` and an `x-data="{ copied: false }"`
             on the enclosing element (code_window sets both up).
  tone     : 'dark' (light-on-dark, for code windows / dark cards) or
             'light' (dark-on-light, for white UI cards).

  code_window parameters
  ----------------------
  title, copyable : forwarded to window_header (tone is always 'dark').

  Example
  -------
    {% import 'macros/code_window.html' as ui_code %}

    {% call ui_code.code_window(title='permits/search', copyable=True) %}
GET /v2/permits/search
    {% endcall %}

    Bespoke light card sharing the same header:
    

      {{ ui_code.window_header('permit record', tone='light') }}
      
…

    


  Notes
  -----
  - `caller() | trim` strips the leading/trailing newline the {% call %}
    block introduces; author intentional code indentation flush-left.
  - Suggested code palette: verbs `text-green-400`, paths `text-white`,
    keys `text-sky-300`, values/numbers `text-amber-300`, comments
    `text-gray-500`.
#}
